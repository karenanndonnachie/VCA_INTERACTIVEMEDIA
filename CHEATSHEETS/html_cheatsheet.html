WHAT IS HTML & HOW TO READ IT

HTML is a structure language. Think of it as nested boxes: a page contains sections, sections contain headings and paragraphs, and each part is wrapped in tags.

Nested Tag Syntax

<main>
  <section>
    <h1>Title</h1>
    <p>This paragraph is inside the section.</p>
  </section>
</main>

Self-Closing / Void Elements

Some elements do not wrap content, so they do not need a closing pair. Common examples are <img>, <br>, <hr>, and <meta>.

<img src="poster.jpg" alt="Film poster">
<br>
<hr>

Where CSS Can Live

  • Inline style: quick one-off styling on one element only
  • <style> block: CSS inside the page, usually in the <head>
  • External stylesheet: linked file like styles.css (best for real projects)
<!-- inline -->
<p style="color: blue;">Inline style example</p>

<!-- in <head> -->
<style>
  p { color: blue; }
</style>

<!-- external -->
<link rel="stylesheet" href="styles.css">

CORE STRUCTURE

Document Setup

Every HTML document begins with a DOCTYPE and basic structure. This tells the browser what kind of document to expect.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
</head>
<body>
  <!-- Content goes here -->
</body>
</html>

Key Meta Tags

  • charset="UTF-8" - Character encoding for text
  • viewport - Responsive design for mobile devices
  • title - Browser tab and search engine display

ACCESSIBILITY BASICS

  • Alt text for images: alt="Descriptive text"
  • Semantic HTML: Use proper heading hierarchy (h1→h2→h3)
  • Labels for inputs: <label for="id">
  • ARIA attributes: aria-label, role
  • Skip links: Allow keyboard navigation to main content
